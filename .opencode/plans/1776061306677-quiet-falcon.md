# Plan: Improve iOS Safari Chrome Continuity

## Goal
Make the site feel visually continuous with iPhone Safari in normal browser-tab mode, knowing Safari owns the real browser chrome and cannot be fully recolored by the page.

## Recommended Approach
Use edge-to-edge viewport behavior plus safe-area-aware page shells so the app background extends cleanly into the notch and home-indicator regions where Safari allows it.

## Why The Current Fix Didn’t Work
- `themeColor` is only a hint in normal iOS Safari tab mode.
- `appleWebApp.statusBarStyle` only matters for Home Screen / standalone launches.
- The app currently has dark metadata and dark root backgrounds, but it does not opt into `viewport-fit=cover` or handle `env(safe-area-inset-*)`.
- Result: Safari still shows its own black top/bottom regions, and the page stops short of feeling edge-to-edge.

## Files To Modify
- `app/layout.tsx`
- `app/globals.css`
- `app/page.tsx`
- `app/about/page.tsx`

## Implementation Steps
1. Update `app/layout.tsx`
Add `viewportFit: "cover"` to the exported `viewport`.
Add `colorScheme: "dark"` to the exported `viewport`.
Keep the existing dark `themeColor`.

2. Update `app/globals.css`
Add reusable safe-area CSS variables using `env(safe-area-inset-top, 0px)` and `env(safe-area-inset-bottom, 0px)`.
Add a shared page-shell class for dark routes that:
- preserves the current background/look
- fills the viewport with iOS-friendly height handling
- adds top and bottom padding from the safe-area variables

3. Update `app/page.tsx`
Replace the current outer/inner `min-h-screen` shell usage with the shared safe-area-aware shell classes.
Do not redesign the content, shader, nav, or footer.

4. Update `app/about/page.tsx`
Mirror the same page-shell changes used on the home page so both routes behave identically on iPhone Safari.

## What I Would Not Change In The First Pass
- `components/site-nav.tsx`
- `components/site-footer.tsx`

Reason: the issue is primarily at the page-shell level. Only adjust nav/footer directly if spacing still feels off after safe-area padding is applied.

## Expected Result
- Better top-edge continuity under the Safari status/address-bar area.
- Better bottom-edge continuity near the home-indicator / toolbar region.
- Slightly increased top and bottom spacing on notched iPhones.
- Safari browser chrome may still remain system-black in places, but the page will feel more intentionally edge-to-edge.

## Risks / Tradeoffs
- Safe-area padding will slightly change the vertical spacing on iPhones.
- Normal Safari browser chrome is still partly outside page control, so this is a polish improvement, not a full recoloring fix.

## Verification
1. Run `npm run check`.
2. Run `npm run build`.
3. Open `/` and `/about` on a real iPhone in normal Safari tab mode.
4. Check portrait first, then landscape if possible.
5. Verify:
- top edge feels visually continuous with the page background
- footer/content do not collide with the bottom toolbar/home-indicator area
- spacing remains acceptable on both routes
6. Scroll so Safari chrome expands/collapses and confirm the result still looks coherent.
