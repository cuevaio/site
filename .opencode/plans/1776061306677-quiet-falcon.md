# Plan: Match Rauch-Style Safari Continuity

## Goal
Make the site behave more like `rauchg.com` on iPhone Safari: one continuous dark canvas with no visible seam between the page background and Safari’s top/bottom regions.

## Key Finding
The main issue is no longer metadata. The site already has dark viewport metadata and safe-area settings.

The remaining mismatch is structural:
- the app still uses multiple independently-sized full-screen effect layers
- some are global in `app/layout.tsx`
- some are page-level
- some use `fixed inset-0`
- some use `100vh`/`100vw`
- the shader uses `contain: strict`

That combination is fragile on iPhone Safari and is likely causing the detached black-band effect.

## What Rauch Does Differently
From inspecting `https://rauchg.com`:
- the page is structurally flat
- the body owns a simple dark canvas
- the main page shell is simple and continuous
- there are no competing decorative fixed layers trying to own the viewport edges
- the site looks like one dark surface, not a content layer sitting above separately-sized background systems

Important note:
- `rauchg.com` even uses `theme-color: transparent`
- so the cleaner Safari result is not coming from stronger metadata tricks
- it is coming from a simpler, flatter page canvas

## Recommended Approach
Refactor the site so each page shell is the single owner of the visible background on mobile, while preserving the richer shader/texture stack on desktop.

Scope confirmed:
- keep the current richer effect treatment on desktop
- simplify only the mobile shell
- optimize specifically for iPhone-sized Safari behavior without flattening the desktop experience

## Files To Modify
- `app/layout.tsx`
- `app/globals.css`
- `components/shader-background.tsx`
- `components/grain-overlay.tsx` or remove its usage
- `app/page.tsx`
- `app/about/page.tsx`

## Implementation Steps
1. Simplify `app/layout.tsx`
Remove the global `app-texture` SVG node.
Remove the global `app-grain` node.
Keep the metadata and viewport exports.

Reason:
The layout should stop painting global full-screen decorative layers for all breakpoints. Desktop-specific visual richness should instead be owned by the route shell so mobile can opt out cleanly.

2. Make the page shell the only background owner in `app/globals.css`
Keep `html` and `body` dark.
Keep safe-area variables.
Update the shared shell so it acts as the single canvas:
- `position: relative`
- dark background color
- safe-area top/bottom padding
- `min-height: 100svh`
- `100dvh` enhancement where supported

Add a dedicated child background layer class for route-level effects, for example:
- absolute inset-0
- pointer-events-none
- z-index below content

Add responsive behavior so:
- mobile uses a flatter background owner with no viewport-bound effect stack
- desktop enables the richer decorative layers

Add a dedicated child content layer class above it.

3. Refactor `components/shader-background.tsx`
Change it from viewport-bound to parent-bound and make it desktop-only by default:
- remove `fixed inset-0`
- use `absolute inset-0`
- remove `contain: strict`
- hide or disable it on mobile breakpoints
- keep the same visual shader settings on desktop unless they need minor tuning later

Reason:
The shader should fill the page shell, not independently size itself against the viewport. On mobile Safari, removing it from the viewport edge path reduces seam risk while preserving the effect on desktop.

4. Consolidate grain/texture ownership
Choose one system only, and make it responsive.

Recommended:
- keep a single grain/texture overlay inside the page shell
- remove the duplicate global-plus-page overlay pattern
- allow subtle texture on desktop while using a flatter, simpler background on mobile

That means either:
- rework `components/grain-overlay.tsx` into a parent-bound absolute layer and keep it
or
- remove it and let the shader + one subtle texture layer handle the look

5. Update `app/page.tsx`
Inside the shell:
- mount one background layer container
- render `ShaderBackground` there for desktop only
- render the single retained grain/texture overlay there with mobile/desktop responsiveness
- keep content in a separate relative layer above it

Do not redesign nav/footer/content layout in the first pass.

6. Update `app/about/page.tsx`
Mirror the same page-shell structure as the home page so both routes use the same Safari-safe canvas pattern.

## What To Avoid
- no global decorative fixed layers in `app/layout.tsx`
- no duplicate grain systems
- no `100vw`/`100vh` sizing on decorative full-screen layers
- no viewport-bound shader root
- no `contain: strict` on the shader wrapper unless proven necessary

## Expected Result
- mobile Safari reads as one flatter dark canvas more like `rauchg.com`
- desktop keeps the richer shader/texture feel
- Safari top/bottom regions should visually blend better on iPhone because the mobile background is structurally unified
- overscroll and toolbar transitions should have fewer seams on mobile

## Tradeoffs
- mobile will look slightly flatter than desktop by design
- removing duplicate/global layers may reduce the current background richness on small screens
- the background system will be simpler and more robust on mobile while keeping the existing visual character on desktop
- some breakpoint tuning may be needed if tablet behavior sits between the two modes

## Verification
1. Run `npm run check`.
2. Run `npm run build`.
3. Test on a real iPhone Safari device in normal browser-tab mode.
4. Compare `/` and `/about` against the current live version.
5. Check:
- initial load
- collapsed and expanded Safari chrome
- top overscroll bounce
- bottom overscroll bounce
- route-to-route consistency
6. Compare directly with `rauchg.com` on the same device for:
- continuity of the top edge
- continuity of the bottom edge
- overall “single surface” feeling

## Recommended First Execution Order
1. Remove global fixed texture/grain from `app/layout.tsx`
2. Make `ShaderBackground` parent-bound and desktop-only
3. Keep only one grain/texture layer with responsive behavior
4. Update both routes to use the unified background-layer/content-layer shell
5. Verify on iPhone Safari first, then confirm desktop still preserves the richer effect stack
